# Ralph Lit Rev

Autonomous structured extraction from academic papers using Claude Code or Codex.

Clone into any project directory containing PDFs. Ralph Lit Rev processes papers one at a time, extracts structured data according to a custom schema, and builds a JSON dataset of your extractions.

## Quickstart

```bash
# Clone into your project
git clone -b ralph-lit-rev <repo-url> ralph

# Set up Python tools
make -C ralph setup

# 1. Design your extraction schema (interactive)
#    Run /schema in Claude Code or Codex

# 2. Index your PDFs
#    Run /prepare in Claude Code or Codex

# 3. Run the extraction loop
make -C ralph run
```

## Workflow

### 1. `/schema` — Design extraction schema

Interactive skill that asks about your review goals and builds `extraction-schema.json`. Guides you through:
- What is the main goal of the review?
- What type of literature? What discipline?
- What aspects matter most?
- Proposes fields, lets you add/remove/modify

### 2. `/prepare` — Index PDFs

Scans a directory for PDF files and builds `articles-meta.json` with one entry per paper.

### 3. `make run` — Extract

Runs the extraction loop. Each iteration:
1. Picks the next unprocessed article
2. Extracts PDF text via pymupdf
3. Extracts structured fields per the schema
4. Appends to `extractions.json`
5. Marks article as processed
6. Logs progress

## Files

| File | Description |
|------|-------------|
| `ralph.sh` | Main loop script |
| `CLAUDE.md` | Core extraction prompt |
| `AGENTS.md` | Core extraction prompt for Codex |
| `extraction-schema.json` | Your extraction fields (generated by /schema) |
| `articles-meta.json` | Your article index (generated by /prepare) |
| `extractions.json` | Output: growing list of extractions |
| `progress.txt` | Extraction log with accumulated patterns |
| `tools/` | Python PDF extraction scripts |
| `skills/` | Slash command definitions for schema/prepare |

## Commands

```bash
make setup          # Create Python venv, install pymupdf
make run            # Run extraction loop (safe mode)
make run N=20       # Limit to 20 iterations
make run-dangerous  # Skip permission checks
make run-codex      # Run extraction loop with Codex
make run-codex-dangerous  # Run Codex without sandbox/approval checks
make init-codex     # Create .codex/config.toml and rules in the project
make install        # Copy skills to .claude/skills and .codex/skills
make status         # Show extraction progress
make progress       # Show progress log
make reset          # Reset all articles to unprocessed
make clean          # Remove all generated artifacts
```

## Requirements

- Claude Code CLI (`claude`) or Codex CLI (`codex`)
- Python 3.10+
- `jq`
